# 委托实际应用场景：回调模式详解

## 一、什么是回调模式？

**回调模式（Callback Pattern）** 是一种编程设计模式，它允许我们将一个方法作为参数传递给另一个方法，当某个操作完成或特定事件发生时，系统会自动调用这个传入的方法。

简单来说：**"你先去做事，做完了告诉我一声"** —— 这就是回调的核心思想。

## 二、代码分析

### 2.1 核心代码结构

```csharp
// 委托声明：定义回调函数的签名
public delegate void DataProcessor(int[] data);

// 异步操作方法：接受回调委托作为参数
static void PerformAsyncOperation(int[] data, DataProcessor callback)
{
    Console.WriteLine("开始异步操作...");

    // 模拟耗时操作（如网络请求、文件读取等）
    System.Threading.Thread.Sleep(1000);

    Console.WriteLine("异步操作完成，执行回调...");

    // 操作完成后，调用回调函数通知调用者
    callback?.Invoke(data);
}
```

### 2.2 使用示例

```csharp
// 调用异步操作，并传入回调函数
PerformAsyncOperation(numbers, (data) =>
{
    Console.WriteLine("异步操作完成！处理结果:");
    CalculateSum(data);  // 在回调中处理数据
});
```

## 三、执行流程详解

让我们逐步分析程序的执行过程：

```
1. 主程序调用 PerformAsyncOperation()
   ↓
2. 输出 "开始异步操作..."
   ↓
3. 执行耗时操作 Thread.Sleep(1000) —— 模拟异步任务
   ↓
4. 输出 "异步操作完成，执行回调..."
   ↓
5. 调用 callback?.Invoke(data) —— 触发回调
   ↓
6. 执行回调函数中的代码：
   - 输出 "异步操作完成！处理结果:"
   - 调用 CalculateSum(data) 计算总和
   ↓
7. 回调执行完毕，返回主程序
```

## 四、关键技术点

### 4.1 Lambda 表达式作为回调

```csharp
(data) =>
{
    Console.WriteLine("异步操作完成！处理结果:");
    CalculateSum(data);
}
```

这是一个 **Lambda 表达式**，它创建了一个匿名方法作为回调函数：
- `(data)` —— 参数列表，接收 int[] 类型的数据
- `=>` —— Lambda 运算符
- `{ ... }` —— 方法体，定义回调时要执行的操作

### 4.2 空值条件运算符 `?.`

```csharp
callback?.Invoke(data);
```

这是 C# 的 **空值条件运算符**：
- 如果 `callback` 不为 null，则调用 `Invoke(data)`
- 如果 `callback` 为 null，则不执行任何操作，避免空引用异常
- 等价于：
  ```csharp
  if (callback != null)
  {
      callback.Invoke(data);
  }
  ```

## 五、回调模式的实际应用场景

### 5.1 异步操作完成通知

```csharp
// 文件下载完成后的回调
DownloadFile(url, (filePath) =>
{
    Console.WriteLine($"文件下载完成: {filePath}");
    ProcessFile(filePath);
});
```

### 5.2 事件处理

```csharp
// 按钮点击事件
button.OnClick += () =>
{
    Console.WriteLine("按钮被点击了！");
};
```

### 5.3 数据库查询回调

```csharp
// 查询完成后处理结果
database.Query("SELECT * FROM Users", (results) =>
{
    foreach (var user in results)
    {
        Console.WriteLine(user.Name);
    }
});
```

### 5.4 Unity 游戏开发中的应用

```csharp
// 动画播放完成回调
PlayAnimation("Jump", () =>
{
    Console.WriteLine("跳跃动画播放完成");
    EnablePlayerControl();
});

// 网络请求回调
NetworkManager.SendRequest(url, (response) =>
{
    UpdateUI(response.data);
});
```

## 六、回调模式的优势

### 6.1 解耦合
- 调用者不需要知道操作的具体实现细节
- 被调用者不需要知道操作完成后要做什么
- 双方通过委托接口进行通信

### 6.2 灵活性
- 可以动态指定操作完成后的行为
- 同一个异步方法可以配合不同的回调使用

```csharp
// 不同的回调处理方式
PerformAsyncOperation(data1, (d) => CalculateSum(d));
PerformAsyncOperation(data2, (d) => CalculateAverage(d));
PerformAsyncOperation(data3, (d) => PrintArray(d));
```

### 6.3 异步编程支持
- 避免阻塞主线程
- 提高程序响应性
- 适合处理耗时操作

## 七、与其他模式的对比

### 7.1 回调 vs 轮询

**轮询方式（不推荐）：**
```csharp
// 不断检查操作是否完成
while (!operation.IsCompleted)
{
    Thread.Sleep(100);
}
ProcessResult(operation.Result);
```

**回调方式（推荐）：**
```csharp
// 操作完成后自动通知
operation.OnCompleted((result) => ProcessResult(result));
```

### 7.2 回调 vs 返回值

**返回值方式（同步）：**
```csharp
int[] result = PerformOperation(data);  // 阻塞等待
ProcessResult(result);
```

**回调方式（异步）：**
```csharp
PerformAsyncOperation(data, (result) => ProcessResult(result));  // 不阻塞
```

## 八、注意事项

### 8.1 回调地狱（Callback Hell）

当回调嵌套过多时，代码可读性会下降：

```csharp
// 不好的示例：多层嵌套
Operation1(data, (result1) =>
{
    Operation2(result1, (result2) =>
    {
        Operation3(result2, (result3) =>
        {
            // 嵌套太深，难以维护
        });
    });
});
```

**解决方案：**
- 使用 async/await（C# 现代异步编程）
- 将回调函数提取为独立方法
- 使用 Promise/Task 链式调用

### 8.2 异常处理

回调中的异常需要妥善处理：

```csharp
static void PerformAsyncOperation(int[] data, DataProcessor callback)
{
    try
    {
        // 执行操作
        callback?.Invoke(data);
    }
    catch (Exception ex)
    {
        Console.WriteLine($"回调执行出错: {ex.Message}");
    }
}
```

### 8.3 内存泄漏风险

长期持有委托引用可能导致内存泄漏，使用完后应及时解除订阅。

## 九、现代 C# 的演进

虽然回调模式很有用，但 C# 提供了更现代的异步编程方式：

### 9.1 async/await 模式

```csharp
// 现代异步写法
async Task<int[]> PerformAsyncOperationAsync(int[] data)
{
    await Task.Delay(1000);  // 异步等待
    return data;
}

// 使用方式
var result = await PerformAsyncOperationAsync(numbers);
CalculateSum(result);
```

### 9.2 Task 延续

```csharp
Task.Run(() => PerformOperation(data))
    .ContinueWith(task => ProcessResult(task.Result));
```

## 十、总结

**回调模式的本质：**
- 将"做什么"和"什么时候做"分离
- 通过委托实现方法的参数化传递
- 是异步编程的基础模式

**适用场景：**
- 异步操作完成通知
- 事件驱动编程
- 插件式架构
- 需要解耦的场景

**学习建议：**
1. 理解委托是回调的基础
2. 掌握 Lambda 表达式的使用
3. 了解现代 async/await 模式
4. 在实际项目中灵活运用

回调模式是理解 C# 异步编程和事件驱动架构的重要基础，掌握它将帮助你更好地理解 Unity、ASP.NET 等框架的设计思想。
