# ScriptableObject 示例代码说明

本文档详细说明每个示例的用途和使用方法。

## 📁 01_Basic - 基础示例

### SimpleData.cs
**用途：** 最简单的 ScriptableObject 实现

**特点：**
- 演示基本的数据存储
- 使用 `[CreateAssetMenu]` 创建资产
- 包含常用的数据类型（string, int, Sprite）
- 使用 `[Header]` 和 `[Tooltip]` 改善编辑器体验

**如何使用：**
1. 在 Unity 中右键 `Create > Tutorial > 01_Basic > Simple Data`
2. 在 Inspector 中填写数据
3. 将创建的资产拖到 `SimpleDataUser` 组件上

### SimpleDataUser.cs
**用途：** 演示如何使用 ScriptableObject 数据

**重要概念：**
- ✅ **正确做法：** 复制 SO 数据到本地变量
- ❌ **错误做法：** 直接修改 SO 数据（会永久改变资产）

**使用步骤：**
1. 创建一个 GameObject
2. 添加 `SimpleDataUser` 组件
3. 分配 `SimpleData` 资产
4. 运行游戏查看控制台输出

## 📁 03_Advanced - 高级示例

### GameEvent.cs + GameEventListener.cs + GameEventTrigger.cs
**用途：** 基于 ScriptableObject 的事件系统

**优势：**
- 完全解耦的通信方式
- 不需要直接引用
- 易于调试和测试
- 支持多个监听者

**使用流程：**

1. **创建事件：**
   - 右键 `Create > Tutorial > 03_Advanced > Game Event`
   - 命名如 "OnPlayerDeath"

2. **添加监听器：**
   - 在 GameObject 上添加 `GameEventListener`
   - 拖入事件资产
   - 在 Response 中添加要执行的方法

3. **触发事件：**
   ```csharp
   // 方法 1：使用 GameEventTrigger 组件
   // 方法 2：在代码中调用
   public GameEvent onPlayerDeath;
   onPlayerDeath.Raise();
   ```

**实际应用示例：**
- 玩家死亡事件 → 显示游戏结束界面
- 关卡完成事件 → 播放胜利音乐、显示奖励
- 敌人击败事件 → 更新任务进度

## 📁 04_Practical - 实战案例

### EnemyData.cs + Enemy.cs
**用途：** 完整的敌人系统

**EnemyData 包含：**
- 战斗属性（生命、攻击、防御）
- AI 行为参数（视野、追击距离）
- 掉落系统（物品、概率）
- 奖励配置（经验、金币）

**Enemy 组件功能：**
- 从 EnemyData 读取配置
- 实现 AI 行为（追击、攻击、逃跑）
- 战斗逻辑（受伤、死亡）
- 掉落生成

**使用方法：**
1. 创建多个 EnemyData 资产（哥布林、兽人、巨龙等）
2. 创建敌人预制体，添加 Enemy 组件
3. 分配对应的 EnemyData
4. 同一个预制体可以通过更换 EnemyData 变成不同敌人

**优势：**
- 100 个哥布林共享 1 份数据
- 策划可以轻松调整数值
- 不需要修改代码

### SkillData.cs + SkillSystem.cs
**用途：** 技能系统

**SkillData 功能：**
- 技能类型（伤害、治疗、增益等）
- 目标类型（单体、群体、区域）
- 冷却和消耗管理
- 等级成长系统

**SkillSystem 功能：**
- 技能使用逻辑
- 冷却时间追踪
- 魔法值管理
- 技能效果执行

**扩展方向：**
- 技能树系统
- 技能组合连招
- 技能升级系统

### LevelData.cs
**用途：** 关卡配置系统

**核心功能：**
- 波次管理（敌人类型、数量、间隔）
- 奖励系统
- 星级评价
- 关卡验证

**使用场景：**
- 塔防游戏的关卡配置
- Roguelike 的房间配置
- 生存模式的波次设置

**工作流程：**
1. 创建 LevelData 资产
2. 配置波次（拖入 EnemyData）
3. 设置奖励和星级条件
4. 使用 "验证关卡配置" 检查错误

## 💡 最佳实践总结

### ✅ 应该做的事

1. **数据与逻辑分离**
   ```csharp
   // ✅ 好
   public EnemyData data;  // 数据
   private int currentHealth;  // 运行时状态
   ```

2. **使用描述性命名**
   ```csharp
   // ✅ 好
   EnemyData.cs, WeaponConfig.cs

   // ❌ 差
   Data.cs, Config.cs
   ```

3. **提供编辑器辅助功能**
   ```csharp
   [ContextMenu("验证数据")]
   void Validate() { }
   ```

4. **使用 Header 和 Tooltip**
   ```csharp
   [Header("战斗属性")]
   [Tooltip("最大生命值")]
   public int maxHealth;
   ```

### ❌ 不应该做的事

1. **运行时修改 SO**
   ```csharp
   // ❌ 永远不要这样做
   enemyData.health -= 10;
   ```

2. **在 SO 中存储运行时状态**
   ```csharp
   // ❌ 不要在 SO 中存储这些
   public bool isAlive;
   public Vector3 currentPosition;
   ```

3. **过度使用单例**
   ```csharp
   // ❌ 不是所有 SO 都需要单例
   // 只有全局唯一的配置才考虑单例
   ```

```csharp
void OnDrawGizmosSelected()
{
    Gizmos.color = Color.yellow;
    Gizmos.DrawWireSphere(transform.position, detectionRange);
}
```

### 3. 数据验证
```csharp
void OnValidate()
{
    if (maxHealth <= 0)
        maxHealth = 100;
}
```
