# Unity ScriptableObject 快速开始指南

## 🚀 5 分钟快速上手

### 第一步：创建你的第一个 ScriptableObject

1. 将 `Examples/01_Basic/SimpleData.cs` 复制到你的 Unity 项目的 `Assets/Scripts` 文件夹

2. 在 Unity 编辑器中，右键点击 Project 窗口
   - 选择 `Create > Tutorial > 01_Basic > Simple Data`
   - 命名为 "MyFirstData"

3. 在 Inspector 中编辑数据：
   - Item Name: "生命药水"
   - Value: 50
   - Description: "恢复 50 点生命值"

### 第二步：使用数据

1. 将 `SimpleDataUser.cs` 复制到项目中

2. 在场景中创建一个空 GameObject，命名为 "DataUser"

3. 添加 `SimpleDataUser` 组件

4. 将 "MyFirstData" 资产拖到组件的 Data 字段

5. 运行游戏，查看控制台输出

**恭喜！你已经完成了第一个 ScriptableObject 的创建和使用！** 🎉

## 💡 常见问题速查

### Q: 为什么我的 ScriptableObject 在运行时被修改了？
**A:** 你可能直接修改了 SO 的数据。应该这样做：

```csharp
// ❌ 错误
public EnemyData data;
void TakeDamage(int damage) {
    data.health -= damage;  // 会永久修改资产！
}

// ✅ 正确
public EnemyData data;
private int currentHealth;

void Start() {
    currentHealth = data.health;  // 复制到本地
}

void TakeDamage(int damage) {
    currentHealth -= damage;  // 修改本地副本
}
```

### Q: 如何在代码中访问 ScriptableObject？
**A:** 三种方式：

```csharp
// 方式 1：Inspector 引用（推荐）
public ItemData itemData;

// 方式 2：Resources.Load
ItemData data = Resources.Load<ItemData>("Items/Sword");

// 方式 3：单例模式（仅用于全局配置）
GameConfig.Instance.masterVolume;
```

### Q: ScriptableObject 和 Prefab 有什么区别？
**A:**
- **ScriptableObject**: 纯数据容器，不能放入场景
- **Prefab**: GameObject 模板，可以实例化到场景

### Q: 可以在 ScriptableObject 中引用场景对象吗？
**A:** 不可以。SO 是资产文件，独立于场景存在。

## 🛠️ 导入到 Unity 项目

### 方法 1：手动复制（推荐新手）
1. 在 Unity 项目中创建 `Assets/ScriptableObjects/Scripts` 文件夹
2. 将 `Examples` 文件夹中的 `.cs` 文件复制进去
3. 等待 Unity 编译完成

### 方法 2：整体导入
1. 将整个 `Examples` 文件夹拖入 Unity 的 `Assets` 文件夹
2. Unity 会自动识别和编译脚本

### 建议的项目结构
```
Assets/
├── ScriptableObjects/
│   ├── Scripts/              # SO 脚本
│   │   ├── Items/
│   │   ├── Enemies/
│   │   └── Skills/
│   │
│   └── Data/                 # SO 资产文件
│       ├── Items/
│       │   ├── Weapons/
│       │   ├── Armors/
│       │   └── Consumables/
│       ├── Enemies/
│       └── Skills/
│
├── Scripts/                  # 其他游戏脚本
└── Scenes/
```
