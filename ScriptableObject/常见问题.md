# ScriptableObject 常见问题与解决方案

## 🔴 常见错误

### 错误 1：运行时修改 ScriptableObject 导致数据永久改变

**问题描述：**
```csharp
public class Enemy : MonoBehaviour
{
    public EnemyData data;

    void TakeDamage(int damage)
    {
        data.health -= damage;  // ❌ 错误！
    }
}
```

**现象：**
- 游戏运行后，敌人的生命值被修改
- 停止游戏后，ScriptableObject 资产文件中的数据也被改变了
- 下次运行游戏时，敌人的初始生命值是错误的

**原因：**
ScriptableObject 是资产文件，直接修改会永久保存到磁盘。

**解决方案：**
```csharp
public class Enemy : MonoBehaviour
{
    public EnemyData data;
    private int currentHealth;  // ✅ 使用本地变量

    void Start()
    {
        currentHealth = data.health;  // 复制数据
    }

    void TakeDamage(int damage)
    {
        currentHealth -= damage;  // 修改本地副本
    }
}
```

### 错误 2：在 ScriptableObject 中引用场景对象

**问题描述：**
```csharp
[CreateAssetMenu]
public class GameData : ScriptableObject
{
    public Transform playerTransform;  // ❌ 错误！
    public GameObject enemy;           // ❌ 错误！
}
```

**现象：**
- 无法在 Inspector 中分配场景对象
- 即使分配了，保存后引用会丢失

**原因：**
ScriptableObject 是资产文件，不能引用场景中的对象。

**解决方案：**
```csharp
// 方案 1：使用标签查找
GameObject player = GameObject.FindGameObjectWithTag("Player");

// 方案 2：使用单例
Player.Instance.transform;

// 方案 3：通过事件通信
public GameEvent onPlayerSpawned;
```

---

### 错误 3：忘记复制 List 或 Array

**问题描述：**
```csharp
public class Inventory : MonoBehaviour
{
    public InventoryData data;

    void Start()
    {
        // ❌ 这只是引用，不是复制！
        List<ItemData> items = data.items;
        items.Add(newItem);  // 会修改原始 SO！
    }
}
```

**解决方案：**
```csharp
void Start()
{
    // ✅ 创建新的 List
    List<ItemData> items = new List<ItemData>(data.items);
    items.Add(newItem);  // 只修改本地副本
}
```

### 错误 4：在 ScriptableObject 中使用 Awake/Start

**问题描述：**
```csharp
[CreateAssetMenu]
public class GameConfig : ScriptableObject
{
    void Start()  // ❌ 不会被调用！
    {
        Debug.Log("Start");
    }
}
```

**原因：**
ScriptableObject 不是 MonoBehaviour，没有 Start/Update 等生命周期。

**解决方案：**
```csharp
[CreateAssetMenu]
public class GameConfig : ScriptableObject
{
    void OnEnable()  // ✅ SO 被加载时调用
    {
        Debug.Log("ScriptableObject loaded");
    }

    void OnDisable()  // ✅ SO 被卸载时调用
    {
        Debug.Log("ScriptableObject unloaded");
    }
}
```

### 错误 5：Resources.Load 路径错误

**问题描述：**
```csharp
// ❌ 错误的路径
ItemData item = Resources.Load<ItemData>("Assets/Resources/Items/Sword.asset");
```

**解决方案：**
```csharp
// ✅ 正确的路径（不包含 Assets/Resources/ 和扩展名）
ItemData item = Resources.Load<ItemData>("Items/Sword");

// 文件实际位置：Assets/Resources/Items/Sword.asset
```

## ⚠️ 常见陷阱

### 陷阱 1：多个对象共享同一个 SO 的副作用

**场景：**
```csharp
// 创建了 10 个哥布林，都引用同一个 GoblinData
// 如果不小心修改了 SO，所有哥布林都会受影响
```

**最佳实践：**
- 始终使用本地变量存储运行时数据
- 只从 SO 读取初始值
- 永远不要写入 SO

---

### 陷阱 2：在编辑器模式下修改 SO

**问题：**
```csharp
#if UNITY_EDITOR
void OnValidate()
{
    // 这会在编辑器中每次修改时触发
    // 可能导致意外的数据修改
}
#endif
```

**建议：**
- 使用 `OnValidate()` 时要小心
- 只用于数据验证，不要修改其他数据
- 添加条件检查避免无限循环

---

### 陷阱 3：序列化问题

**不能序列化的类型：**
```csharp
[CreateAssetMenu]
public class BadData : ScriptableObject
{
    public Dictionary<string, int> dict;  // ❌ 不会被序列化
    private int value;                     // ❌ private 不会被序列化
    public Action callback;                // ❌ 委托不能序列化
}
```

**解决方案：**
```csharp
[CreateAssetMenu]
public class GoodData : ScriptableObject
{
    // ✅ 使用可序列化的类
    [SerializeField]
    private List<KeyValuePair> pairs;

    // ✅ 使用 SerializeField
    [SerializeField]
    private int value;

    // ✅ 使用 UnityEvent
    public UnityEvent onEvent;
}

[System.Serializable]
public class KeyValuePair
{
    public string key;
    public int value;
}
```

## 🐛 调试技巧

### 技巧 1：使用 ContextMenu 快速测试

```csharp
[CreateAssetMenu]
public class EnemyData : ScriptableObject
{
    public int health = 100;

    [ContextMenu("测试：减少生命值")]
    void TestReduceHealth()
    {
        Debug.Log($"当前生命值: {health}");
        // 注意：这会真的修改 SO！
    }

    [ContextMenu("重置数据")]
    void ResetData()
    {
        health = 100;
        Debug.Log("数据已重置");
    }
}
```

**使用方法：**
- 在 Inspector 中右键点击 ScriptableObject
- 选择菜单项执行方法

### 技巧 2：添加数据验证

```csharp
[CreateAssetMenu]
public class ItemData : ScriptableObject
{
    public int buyPrice = 100;
    public int sellPrice = 50;

    void OnValidate()
    {
        // 确保卖价不高于买价
        if (sellPrice > buyPrice)
        {
            Debug.LogWarning($"{name}: 卖价不能高于买价！");
            sellPrice = buyPrice / 2;
        }

        // 确保价格为正数
        if (buyPrice < 0) buyPrice = 0;
        if (sellPrice < 0) sellPrice = 0;
    }
}
```

## 🔧 性能优化

### 优化 1：避免频繁的 Resources.Load

**❌ 低效：**
```csharp
void Update()
{
    // 每帧都加载！
    ItemData item = Resources.Load<ItemData>("Items/Sword");
}
```

**✅ 高效：**
```csharp
private ItemData cachedItem;

void Start()
{
    // 只加载一次
    cachedItem = Resources.Load<ItemData>("Items/Sword");
}

void Update()
{
    // 使用缓存
    if (cachedItem != null)
    {
        // ...
    }
}
```

---

### 优化 2：使用 Addressables（大型项目）

```csharp
using UnityEngine.AddressableAssets;

public class ItemLoader : MonoBehaviour
{
    async void LoadItem()
    {
        // 异步加载，不阻塞主线程
        var handle = Addressables.LoadAssetAsync<ItemData>("Sword");
        ItemData item = await handle.Task;

        // 使用 item
    }
}
```

---

### 优化 3：避免在 SO 中存储大量数据

**❌ 不推荐：**
```csharp
[CreateAssetMenu]
public class LevelData : ScriptableObject
{
    // 存储整个关卡的地形数据
    public int[,] terrainData = new int[1000, 1000];  // 太大了！
}
```

**✅ 推荐：**
```csharp
[CreateAssetMenu]
public class LevelData : ScriptableObject
{
    // 只存储配置，实际数据从文件加载
    public string terrainDataPath;
    public Vector2Int mapSize;
}
```

## 📋 检查清单

在发布游戏前，检查以下事项：

### 数据完整性
- [ ] 所有 SO 资产都已正确配置
- [ ] 没有空引用（null references）
- [ ] 数值在合理范围内
- [ ] 所有必需的资源（图片、音效）都已分配

### 代码质量
- [ ] 没有直接修改 SO 的代码
- [ ] 运行时数据使用本地变量
- [ ] 正确处理了 List/Array 的复制
- [ ] 添加了必要的空值检查

### 性能
- [ ] 没有在 Update 中加载资源
- [ ] 使用了对象池（如果需要）
- [ ] 大型数据使用了异步加载

### 调试
- [ ] 移除或禁用了调试代码
- [ ] 移除了测试用的 ContextMenu
- [ ] 清理了无用的 Debug.Log

### 话题 3：版本控制

**Git 配置建议：**

```gitignore
# .gitignore

# 不要忽略 SO 资产文件
!*.asset

# 但可以忽略自动生成的
*_Generated.asset
```

**团队协作：**
- SO 文件是文本格式（YAML），适合版本控制
- 合并冲突时，优先保留数值更改
- 使用描述性的提交信息

## 💡 最佳实践总结

1. **永远不要在运行时修改 ScriptableObject**
2. **使用本地变量存储运行时状态**
3. **为 SO 添加验证和调试功能**
4. **合理组织文件结构**
5. **使用 ContextMenu 方便测试**
6. **添加详细的注释和文档**
7. **定期备份重要的 SO 资产**
